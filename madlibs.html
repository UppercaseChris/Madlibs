Let's Play A little Game!

Explanation / Walthrough
Well this project was a fun one. Starting off I generally externally linked my work from other coding languages so I decided to swtich things up a bit and code this one right into the html document. I had a few different approaches towards this, most of them using if statements to try and toggle the various coniditons. However after a little trial and error and I found this to be the best method. I went a little bit against in the grain in terms of the buttons. As I'm sure you noticed I used a toggle system of the buttons. When the submit button is clicked it's visibility goes to hidden while the the go back button becomes visable and vice versa. That just felt like a more dynamic approach towards the project. I used two functions for this. One for when the submit button was clicked, and another for when the go back button was clicked. This acted as my toggle system. The goBack button function simply reswapped the visibility of the buttons and wrote some empty innerHTML code to remove what was originally there. The variables themselves would be overwritten when the submit button was clicked for a second time so I didn't bother trying to nullify the variables. I also taken the liberty of adding some HTML in the when writing in the innerHTML in the submit button function to include some bold tags. This was just to emphasis what characters being displayed are the ones the user has entered. I used the toLowerCase method in this innerHTML string accompanying the variables as they were placed to convert all the answers over to string. I didn't include a method of verification so if the input boxes are blank the function will still run, however I imagine it would be a simple if statement, something like
